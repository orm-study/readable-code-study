## 1. 뇌 메모리 적게 쓰는 방법

> 뇌는 효율적으로 동작하기 위해 대상의 특징만 묶어 범주화(일반화)하고 사람은 한 번에 한 가지 일 밖에 못한다.

라는게 이번 장을 시작하면서 한 첫 번째 주장인데 사실 이 내용도 일반화일 가능성이 높아보였다. 뇌는 유기적으로 변하는 대상이고 굉장히 복잡한 기관인데 저렇게 일관되게 설명한다는 것, 아직 완전히 연구되지 않은 대상에 대해 확신있게 얘기하는데 책 두권을 근거로 든다는 부분에서 신빙성을 얻지는 못했다.

## 2. 리턴 빨리해라

```
if (조건1) {
  .. 복잡하고 긴 구문
} else if (조건2) {
  .. 복잡하고 긴 구문
} else {
  ..
}
```

> 위와 같은 코드를 읽을 때 대체로 if와 else if, else를 모두 읽어야 하는 느낌을 받는 경우가 많다고 한다.

근데 각 조건에 수행해야 하는 동작이 길어지는 경우에 모든 절차를 이해하는 동안 각 조건에 대한 내용들을 계속 기억하는 노력을 하고 있어야한다.
그래서 else에 뒤 처리 로직이 모두 몰려있는 경우라면 else if를 사용하지 말고 빨리 리턴해버리라는게 이번 절의 조언이었다.

```
if (조건1) {
  .. 복잡하고 긴 구문
  return; // 바로 리턴해버려서 뒤를 볼 필요 없게
}

if (조건2) {
  .. 복잡하고 긴 구문
  return; // 바로 리턴해버려서 뒤를 볼 필요 없게
}

.. 기존 else의 내용은 그냥 여기 넣어버리면 된다.

```

## 3. 사고의 복잡성을 줄이는 방법 (depth 줄이기)

> a. 이중 for문, 이중 if문의 경우
- 중첩되는 각 구문을 메서드와 하면 생각에 필요한 에너지를 줄일 수 있다.
- 단, 이중 구문만 본다고 무작정 달려들어서 고칠 필욘 없고 이해가 그럭저럭 된다면 그대로 둬도 된다.

> b. 사용할 변수는 가깝게 둬라
- 사용할 변수를 너무 멀리 두면 필요할 때 바로 기억 못하거나 못찾고 `컨트롤 + f`를 해야할 수도 있다.
- 근데 솔직히 이 비용이 뭐 그렇게 큰가 싶기도 하고 여러 곳에서 쓰이는 변수는 어떻게 하나 싶기도 해서 실용적인건지 잘 모르겠다.
